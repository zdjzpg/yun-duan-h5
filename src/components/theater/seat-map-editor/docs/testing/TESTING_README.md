## 📋 测试前准备

### **环境要求**
- ✅ 浏览器：Chrome/Edge/Firefox 最新版本
- ✅ 屏幕分辨率：1920x1080 或更高
- ✅ 键盘：支持 Ctrl/Cmd 快捷键
- ✅ 鼠标：支持左键、右键、滚轮

### **测试数据准备**
```typescript
// 使用测试数据生成器
import { createTestTheaterData, createLargeScaleTestData } from './TEST_DEMO_DATA';

// 基础测试数据（5x5 座位）
const testData = createTestTheaterData();

// 大规模测试数据（20x20 = 400 座位）
const largeData = createLargeScaleTestData(20, 20);
```

### **清空浏览器缓存**
- 清空 LocalStorage
- 清空浏览器缓存
- 硬刷新页面（Ctrl+Shift+R）

## 🎯 测试重点

### **必须测试的核心功能**
1. ✅ **单座位工具** - 添加和编辑单个座位
2. ✅ **批量框选** - 矩形框选多个座位
3. ✅ **座位拖拽** - 拖拽移动座位
4. ✅ **对齐工具** - 6 种对齐方式
5. ✅ **撤销重做** - 正确回退和恢复操作

### **需要重点关注的场景**
1. ⚠️ **大量座位性能** - 400+ 座位时的流畅度
2. ⚠️ **撤销重做边界** - 连续撤销 50 次
3. ⚠️ **批量操作** - 同时拖拽 100+ 座位
4. ⚠️ **快捷键冲突** - 快捷键是否与浏览器冲突

### **常见问题检查**
- [ ] 控制台无错误和警告
- [ ] 没有"Maximum update depth"错误
- [ ] 拖拽时无明显卡顿
- [ ] 座位坐标正确更新
- [ ] 历史记录不超过 50 条

## 📊 测试指标

### **功能完整性**
- **目标：** 所有核心功能正常工作
- **标准：** 通过率 ≥ 95%
- **评分：** 通过/失败

### **性能表现**
- **目标：** 流畅的用户体验
- **标准：**
  - 100 座位：所有操作 < 100ms
  - 400 座位：所有操作 < 300ms
  - 1000 座位：所有操作 < 500ms
- **评分：** 优秀/良好/需优化

### **用户体验**
- **目标：** 直观易用
- **标准：**
  - 学习曲线 < 15 分钟
  - 快捷键响应及时
  - 交互反馈清晰
- **评分：** 1-10 分

### **稳定性**
- **目标：** 无崩溃，无数据丢失
- **标准：**
  - 无控制台错误
  - 连续操作 100 次无崩溃
  - 数据保存正确
- **评分：** 稳定/不稳定

## 🐛 问题报告

### **如何报告问题**

1. **使用测试报告模板**
   - 复制 `TEST_REPORT_TEMPLATE.md`
   - 填写问题详情

2. **问题信息包含**
   - 问题描述
   - 复现步骤（详细）
   - 预期结果 vs 实际结果
   - 优先级（P0-P3）
   - 截图或控制台日志

3. **优先级定义**
   - **P0** - 严重问题，阻塞使用
   - **P1** - 重要问题，影响体验
   - **P2** - 一般问题，轻微影响
   - **P3** - 建议优化，可选
